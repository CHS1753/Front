IP의 역할 (Internet Protocol)

    - 인터넷에 연결된 Host를 구별할 수 있는 주소 제공
    - IP 주소는 국제적으로 관리
    - 장비를 Internet에 연결하려면 당연히 IP가 필요하다.
    - IP가 많이 필요하다보니 공유기를 이용하여 IP를 받는다.


TCP
    - 인터넷에서 각종 제어를 담당
    - 그 중 port라는 개념
    - IP는 Host를 구별
    - 식별하는건 ID
    - TCP port는 Host에서 실행되는 여러 program을 구별한다. (0 ~ 65535)

방화벽 (firewall)
    - 가장 간단한 개념으로 허용되지 port를 모두 막는다.

    - 남의 컴퓨터를 원격으로 사용하려면 IP와 port를 알아야함 하지만 방화벽에 의해 먼저 컷 당한다.
    - 192.168.0.100:1521(oracle)

인터넷에 접속한다? IP와 Port가 필요함

웹 (web)
    - Internet 위에서 HTTP 아니면 HTTPS Protocol을 이용해 HTML 문서를 주고받는 Service
    - 인트라넷? 아니면 군대 메일 시스템
    - FTP -> 파일 보내는 서버, SMTP -> 메일 보내는 서버
    - HTTP는 80번, HTTPS는 443번 port를 사용
      인증: 신원을 확인(보통 로그인으로 서버가 사용자를 인증)
      HTTPS는 사용자 측에서 서버가 인증된 서버인지 확인하는 양방향 인증을 수행
      상호인증 -> 서버와 사용자를 인증하는 것
